Feature foundation

Make scan snapshots durable, portable, and safe to reopen.

Export should preserve Radix's core advantage: fast navigation over an immutable scan tree. Import should never mutate disk, never silently trust stale paths, and never need a rescan to show the original result.

Principles

Export/import should act like another view of scan state, not like backup software or a file action.

Snapshot first

Core

Archive stores a normalized ScanSnapshot graph: target, nodes, topology, warnings, stats, timestamps, and scan options fingerprint.

  • No dependency on current filesystem to render.
  • No implicit rescan on import.
  • No separate UI-only snapshot model.

Trust nothing live

Safety

Imported paths are historical facts. File actions need fresh validation before opening, revealing, or trashing anything.

  • Move to Trash disabled for imported snapshots by default.
  • Reveal and Open require path existence check.
  • Identity match upgrades imported nodes to live-safe actions.

Designed to evolve

Format

v1 should be strict enough for reliable round trips, loose enough for future diffing, comparison, tags, comments, and cloud handoff.

  • Explicit formatVersion; v1 imports reject unsupported future versions before decoding the v1 body.
  • Unknown future fields stay out of v1's compatibility promise until a migration path exists.
  • Import errors point to manifest, node, or topology failure.

User Flows

First release can be small: export current complete scan, import a saved scan, reopen it in workspace.

1

Command: File > Export Snapshot...

Enabled when scanState.snapshot?.isComplete == true. Save panel defaults to target name plus scan date.

2

Use v1 fidelity

v1 keeps absolute paths and file identities. Shareable redaction is future scope.

3

Write archive

Stream manifest, topology, node records, warnings, and aggregate stats. Validate read-back checksum before success toast.

xInclude absolute paths for local reopen.
xInclude scan warnings and permission-limited paths.
xInclude auto-summary nodes exactly as displayed.
xUse deterministic order for stable tests and future diffs.
1

Command: File > Import Snapshot...

Open panel accepts .radixscan. Import parser validates manifest before loading large node payload.

2

Preview summary

Show target, scan date, app version, node count, total size, warnings, path privacy mode, format version.

3

Open snapshot

Call scanCoordinator.restoreCompletedSnapshot with imported snapshot and an imported-source marker.

Imported snapshots are inert. Radix can browse them immediately, but destructive actions stay unavailable until live path and identity validation proves selected nodes still match disk.

Recent Exports

Store file bookmarks, not archive contents, in app preferences. Missing files show stale state.

Open in Place

Future Compare

v2

Snapshot archive should make diffing straightforward: same target path, comparable root identity, deterministic node IDs, stable child ordering.

  • Added, removed, and size-changed paths.
  • Directory growth ranking.
  • Compare live scan against imported baseline.

Do Not Build Yet

Scope

Avoid a diff UI in v1. Build archive data shape so compare can arrive later without format churn.

Node Record v1

Round trip
  • Required: id, path, name, isDirectory, allocatedSize, logicalSize.
  • Preserve: symbolic link, package, accessibility, synthetic, auto-summarized, descendant file count.
  • Preserve hard-link dedupe fields: unduplicatedAllocatedSize, fileIdentity, linkCount.
  • Store lastModified as nullable ISO 8601.

Topology v1

Important
  • Keep childIDsByID order so file browser and sunburst match original scan.
  • Rebuild parentIDByID on import, then validate against supplied parent list if present.
  • Reject duplicate node IDs outside documented repair mode.
  • Repair materialized directory totals using existing FileTreeStore initializer behavior.

Trust Model

Imported scans should feel useful immediately, while Radix remains honest about historical vs live data.

Historical Data

Always allowed
  • Chart navigation.
  • File browser sorting and search.
  • Inspector summary and largest children.
  • Copy archived path when path mode allows.

Live Actions

Guarded
  • Reveal/Open require current path exists.
  • Optional identity check compares device/inode or resource identifier.
  • Mismatch shows "File changed since snapshot" state.
  • Trash remains disabled unless explicitly supported later.

Shareable Mode

Privacy
  • Redact user home prefix to ~ or named root.
  • Optionally strip file identities.
  • Disable live file actions.
  • Still preserve sizes, hierarchy, warnings, summary semantics.

Architecture

Implementation should live mostly in RadixCore, with AppModel coordinating panels and restore behavior.

Proposed Types

RadixCore
  • ScanArchiveDocument: top-level Codable DTO for manifest-level data.
  • ScanArchiveNodeV1: stable exported form of FileNodeRecord.
  • ScanArchiveTopologyV1: root ID and child ID map.
  • ScanArchiveService: async export/import boundary.
  • ImportedSnapshotContext: source URL, path mode, live-action capability.

App Integration

SwiftUI
  • RadixCommands: add Export Snapshot and Import Snapshot to File menu.
  • AppModel: present save/open panel, call service, restore snapshot.
  • ScanCoordinator: retain source kind, live vs imported.
  • FileNodeActionAvailability: gate actions by imported snapshot trust state.
  • RadixCoreTests: round-trip, unsupported version, corrupt archive, large topology tests.
Strongest boundary: ScanArchiveService.export(snapshot:to:options:) and ScanArchiveService.import(from:) never know about SwiftUI panels. App layer owns user choice; core layer owns format correctness.

Roadmap

Build small, test hard, leave enough structure for future snapshot workflows.

Phase 1

Core archive round trip

Codable DTOs, v1 schema, deterministic encode order, import validation, tests against fixtures.

Phase 2

File menu export/import

Save/open panels, progress state, restore imported snapshot into existing workspace, clear errors.

Phase 3

Trust-aware file actions

Imported snapshot source state, live path validation, action gating, stale path messaging.

Phase 4

Snapshot library and compare prep

Recent archives, metadata preview, future shareable mode, diff-ready archive invariants.

Definition of Done

v1
  • Export current complete snapshot, import it, chart/table/inspector match original.
  • Round-trip tests cover hard links, inaccessible nodes, synthetic nodes, auto summaries.
  • Corrupt or unsupported archive fails with actionable error.
  • Imported snapshot cannot mutate files by accident.
